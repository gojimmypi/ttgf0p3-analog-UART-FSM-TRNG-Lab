# Tools

## Tool descriptions

| File                          | Description                                                                                                                                                                                                                                        |
| ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `build_custom_gds_verilog.py` | Expands the split RTL under `src/` into one self-contained Verilog file for the Tiny Tapeout `custom_gds` submission path. This is release-critical because the custom-GDS action expects a single Verilog file.                                   |
| `build_custom_yosys.sh`       | Optional local developer helper for building a pyosys-enabled Yosys tree. This is not used by CI or the normal release flow.                                                                                                                       |
| `check_gds_content.py`        | Validates that the submitted GDS contains real project layout geometry, not just the analog frame/template. With optional checks enabled, it also verifies the known-good ua[5] passive probe geometry.                                            |
| `create_mag.sh`               | Local helper that creates the analog frame Magic layout from the GF180 analog DEF template and applies the Magic-level analog patch. This is a frame/debug helper, not the preferred final GDS generation path.                                    |
| `export_magic_gds_lef.sh`     | Local helper that exports GDS/LEF from the Magic analog frame layout. It intentionally refuses to run unless `ALLOW_FRAME_ONLY_GDS=1` is set, because it does not generate the final hardened RTL-based project GDS.                               |
| `full_harden_artifact.sh`     | CI-oriented hardening script used to regenerate, post-process, check, and stage hardened GDS/LEF artifacts for workflow upload.                                                                                                                    |
| `full_harden_local.sh`        | Local hardening script used by `generate_analog_gds.sh` to run the full Tiny Tapeout hardening flow and copy the resulting GDS/LEF into the repository `gds/` and `lef/` directories.                                                              |
| `generate_analog_gds.sh`      | Preferred local command for regenerating the final analog GDS/LEF from the latest RTL. It runs RTL diagnostics, invokes the local full-harden flow, and validates the final analog outputs.                                                        |
| `patch_analog_mag.py`         | Patches the generated Magic analog frame layout with project-specific power-port information while leaving TT analog pin geometry unchanged. Used by `create_mag.sh`.                                                                              |
| `patch_analog_outputs.py`     | Post-processes generated GDS/LEF outputs by adding corrected project power-port geometry and analog-pin support structures while removing known-bad experimental geometry. Intended to be idempotent.                                              |
| `patch_full_harden_config.py` | Updates Tiny Tapeout hardening configuration for the GF180 analog 1x2 template, including analog DEF selection, die area, power pin names, routing layer, density, and clock period.                                                               |
| `patch_full_harden_source.py` | Temporarily patches `src/project.v` so the normal Tiny Tapeout hardening flow accepts the analog design. It removes top-level `VGND`/`VDPWR` ports only for hardening; callers back up and restore the source afterward.                           |
| `README.md`                   | Documents the purpose and ownership of the scripts in this directory, including which scripts are release-critical, local regeneration helpers, or frame/debug-only utilities.                                                                     |
| `refresh.sh`                  | Compatibility wrapper for older commands. It now delegates to `generate_analog_gds.sh`, so the refresh path uses the full hardened RTL-based GDS flow instead of the older frame-only path.                                                        |
| `strip_unused_analog_pins.py` | Removes disabled `ua[6]` and `ua[7]` GDS text labels after hardening while preserving the LEF pin blocks required by Tiny Tapeout checks. This keeps the physical analog pin labeling aligned with `info.yaml` while maintaining TT compatibility. |
